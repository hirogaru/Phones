
Представления:
MainWindow - класс-представление. Реализует главное окно приложения. Создаёт контроллер ContactController, запускается при старте приложения.
ContactsWindow - окно, запускаемое контроллером ContactController. Отображает в таблице данные, содержащиеся в модели ContactModel. Содержит кнопку добавления контакта, форму поиска и контекстное меню для выделенного контакта.   
AddWindow - представление для добавления или редактирования существующего контакта.

ContactController - реализация контроллера на основе интерфейса , включающего 4 метода. ShowContactsView - метод, позволяющий отображать представление ContactsWindow. ShowSearchResult - метод, помещающий в модель данные соответствующие параметрам поиска. AddContact - метод для добавления или обновления существующего контакта, также отображает представление AddWindow. DeleteContact - метод для удаления контакта из базы (реализован soft delete).

ContactsModel - модель для использования представлением ContactsWindow. Предствление получает обновления из модели и через контроллер передаёт данные обратно в модель. Класс реализует интерфейс IContactsModel для возможности лёгкой расширяемости и тестируемости приложения. Contacts - свойство содержащее коллекцию контактов. ContactUpdated - событие для уведомления экземпляра класса ContactsWindow об обновлении.

SearchEngine - класс для поиска подстроки в полях базы. Метод CleanContent убирает разделители и элементы regex, а метод Search возвращает список совпадений.

SqlRepository. Чтобы отделить модель данных от конкретной реализации, т.е. чтоб контроллер про context и System.Data.Linq ничего не знал будем использовать репозиторий. Класс реализует доступ к данным, а также методы для создания, изменения и удаления этих данных. Реализует интерфейс IRepository.

App - класс приложения, который содержит метод ConfigureContainer необходимый для конфигурации контейнера зависимостей.
